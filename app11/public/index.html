API REST com PostgreSQL

Documentação da API de gerenciamento de usuários com autenticação JWT.

Autenticação

POST /api/auth/register

Registrar novo usuário

Corpo da Requisição:

{
    "name": "Nome do Usuário",
    "email": "usuario@email.com",
    "password": "senha123"
}

POST /api/auth/login

Autenticar usuário

Corpo da Requisição:

{
    "email": "usuario@email.com",
    "password": "senha123"
}

GET /api/auth/profile

Obter perfil do usuário autenticado

Requer: Token JWT no header Authorization

Usuários

GET /api/users

Listar todos os usuários (apenas admin)

Requer: Token JWT de admin

GET /api/users/:id

Obter usuário por ID

Requer: Token JWT (admin ou próprio usuário)

PUT /api/users/:id

Atualizar usuário

Requer: Token JWT (admin ou próprio usuário)

Corpo da Requisição:

{
    "name": "Novo Nome",
    "email": "novo@email.com",
    "role": "user",
    "active": true
}

DELETE /api/users/:id

Excluir usuário

Requer: Token JWT de admin

Autenticação

Para endpoints protegidos, inclua o token JWT no header:

Authorization: Bearer seu_token_jwt

Códigos de Status

Código Descrição
200 Sucesso
201 Criado com sucesso
400 Requisição inválida
401 Não autorizado
403 Proibido
404 Não encontrado
500 Erro interno do servidor
